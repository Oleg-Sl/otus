Скрипт log_analyzer.py

Для корректной работы необходимо создать директорию с файлами лога - "log" и поместить в нее фаил с логами. 
Или передать путь к директории с файлами лога через параметр командной строки:
	python log_analyzer.py --config <путь к файлу конфига>
Фаил конфигурации должен иметь  название "config.conf" и иметь следующую структуру:
REPORT_SIZE: <количество записей в отчете>
REPORT_DIR: <путь к директории хранения отчетов>
LOG_DIR: <путь к директории с файлами лога>

Файлы анализируемого лога должны иметь название (изменяться может только дата):
	"nginx-access-ui.log-20150629"
или
	"nginx-access-ui.log-20150629.gz"

Шаблон файла отчета должен храниться в текущей директории.

Фаил с логами выполнения скрипта пишется в папке текущей директории - "log_log_analyzer"

Отчет хранится в папке текущей директории - "reports"

Запуск тестирования функций log_analyzer.py:
	python test.py
